🔐 What is SECRET_KEY?
The SECRET_KEY is used to sign JWT tokens. Think of it like:
Your SECRET_KEY = Your digital signature
JWT Token = A document signed with your signature

If someone has your signature → They can forge documents!
Example:
python# Your code
SECRET_KEY = "your-secret-key-change-this-in-production"

# Attacker sees this in GitHub
attacker = "I know the secret key!"

# Attacker can create fake tokens
fake_token = jwt.encode(
    {"sub": "user123"},  # Pretend to be user 123
    "your-secret-key-change-this-in-production",  # They know this!
    algorithm="HS256"
)

# Your backend verifies the token
payload = jwt.decode(fake_token, SECRET_KEY)
# ✅ Token validates! (because signature matches)

# Attacker successfully logged in as user 123! 🚨

❌ The Problem
Your default SECRET_KEY:
pythonSECRET_KEY = "your-secret-key-change-this-in-production"
Issues:

❌ It's in your code (everyone can see it in GitHub!)
❌ It's the same for everyone (no uniqueness)
❌ It's weak (not random enough)
❌ Anyone can forge tokens and pretend to be any user
